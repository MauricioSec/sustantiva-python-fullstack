1. ¿Cuál fue la diferencia principal que notaste entre for y while?
La diferencia radica en la certidumbre de la iteración. El ciclo 'for' resultó ideal para recorrer estructuras donde la cantidad de datos ya está definida de antemano (como los elementos de una lista o un rango numérico específico). En cambio, el 'while' es indefinido; se mantiene en ejecución mientras una condición lógica siga siendo verdadera, lo cual es útil cuando no sabemos exactamente cuántas veces debe repetirse el código (por ejemplo, hasta que el usuario decida ingresar un 0).

2. ¿Te pareció útil el uso de break y continue? ¿Por qué?
Sí, resultan fundamentales para tomar el control del flujo del programa en tiempo real. 
'break' es el mecanismo de seguridad que permite abortar un proceso inmediatamente, lo que evita quedar atrapado en un ciclo infinito si cambian las condiciones. 
'continue' me pareció especialmente eficiente para filtrar información; si estoy analizando un listado de datos y encuentro uno que no cumple mis requisitos (como el nombre "Juan" en el ejercicio), 'continue' permite saltarlo e ir al siguiente registro sin tener que usar bloques 'else' que ensuciarían el código.
